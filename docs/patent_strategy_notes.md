# 특허 전략 메모

## 문서 성격

- 이 문서는 ChatGPT, Claude 등 여러 보조 도구가 공용으로 참고하는 최신 전략 메모다.
- 대화별 맥락보다 이 문서의 최신 합의 사항을 우선한다.
- 초안 검토용 판단 기준과 특허 방향성만 간결하게 유지한다.

## 현재 핵심 발명 정의

- 사용자의 머리, 청각계 또는 상체 보호를 위한 착용형 보호장구류에 일체화되거나 결합 가능한 인공지능 기반 극한소음 대응 음성 통신 장치 및 방법
- 특히 보호용 헬멧과의 일체 구성을 주요 아이덴티티로 유지하되, 연산 모듈이 조끼, 복부 보호대, 또는 휴대형 장착체에 분산 장착되는 구성도 허용 가능한 변형예로 본다
- 여기서 휴대형 장착체는 보호장구류 그 자체라기보다 보호장구류와 함께 운용되는 휴대형 장착 수단으로 이해한다
- 기술적 핵심 수단은 공기전도/골전도 멀티모달 입력, 포화 상태 정보, 적응형 연산 제어, 및 포화 대응형 채널 융합에 둔다
- 핵심 가치는 "정확 복원"보다 "음성 명료도와 통신 연속성 확보"에 둔다

## 검토 전제

- 명세서 초안의 비교용 마크업(`reviewpair`, `[기존]`, `[수정안]`)은 현 단계에서 의도적으로 유지할 수 있다
- 따라서 현재 단계 검토에서는 비교용 마크업 존재 자체를 결함으로 보지 않는다
- 명세서 방향성 검토에서는 코드와의 1:1 일치 여부보다 명세서 내부 논리와 기술 서술의 자연스러움을 우선 본다

## 기술적 입장

### 포화 구간 복원 표현

- 공기전도 채널이 포화되고 골전도 채널이 유효한 경우에는, 골전도 채널 정보를 우세하게 활용하여 음성 정보를 재구성하는 방향이 타당하다
- 양 채널이 동시에 길게 포화되어 원신호 정보가 실제로 소실된 경우에는 "원래 발화를 정확히 복원한다"는 표현을 피한다
- 양 채널 동시 포화는 "복원"의 문제보다 "graceful degradation" 또는 "fail-safe 대응"의 문제로 보는 편이 자연스럽다
- 매우 짧은 동시 포화에서는 직전 유효 구간 유지, 짧은 스펙트럼 보간, 또는 comfort noise 유사 처리로 청취상 급격한 이질감을 완화할 수 있다
- 여기서 `자연스러운 연결`은 직전 유효 구간의 단기 유지 또는 스펙트럼 보간 등에 의해 청취 연속성을 급격히 해치지 않도록 처리하는 정도를 뜻한다
- 길거나 강한 동시 포화에서는 모호한 음성 출력을 억제하고, 레벨 감쇠, 자연스러운 페이드아웃, 제한된 배경 소음 유지, 무음, 또는 경고로 전환하는 방향이 안전하다
- 따라서 양 채널 동시 포화에 대해서는 "복원"보다 "자연스러운 연결 또는 감쇠/무음/경고 처리"로 요약하는 편이 적절하다
- 이 경우 허용되는 방향은 보간, 감쇠, 무음, 경고, 또는 청취 연속성 유지 수준의 처리다

### 생성형 모델/LLM

- 생성형 모델이나 LLM은 메인 실시간 통신 경로의 핵심 요소로 두지 않는다
- 메인 경로는 `골전도 + 공기전도 + 포화 상태 정보 + 적응적 융합 + 대체 처리 경로` 구조로 설명하는 것이 안전하다
- 생성형 요소는 넣더라도 보조 계층 또는 비실시간 후속 처리로 한정한다

### 포화 상태 정보와 판단 보조 정보

- 현재 핵심 개념은 `포화 상태 정보`이며, 대표 실시예는 `포화 마스크`다
- 포화 상태를 판단하기 위해 추가 정보가 필요할 수 있으나, 이를 위해 `센서 신뢰도 맵`과 같은 새 중심 용어를 전면에 세우지는 않는다
- 포화 상태 판단에 참고할 수 있는 보조 정보 예시는 아래와 같다
  - 클리핑 비율
  - 포화 지속시간 또는 회복 시간
  - 파형 평탄화 또는 비선형 왜곡 정도
  - 충격성 이벤트 여부
- 국소 에너지, SNR, 또는 골전도 접촉 상태와 같은 값은 직접적인 포화 정보라기보다 채널 상태 참고정보로만 약하게 다룬다
- 청구항 관점에서는 `포화 상태 정보`를 상위 개념으로 두고, 보조 판단 정보는 상세한 설명 또는 종속항에서 필요 시 보강하는 구조가 우선 권장된다

### 전장 적용성 표현

- "전장에서는 항상 마이크가 포화된다"는 식의 일반화는 피한다
- 전장 특수성은 아래 기술 문제로 번역해 서술한다
  - 충격성 소음
  - 공기전도 채널 포화 및 회복 지연
  - 짧은 실시간 지연 요구
  - 오복원이 허용되지 않는 운용 환경

## 모드 설계 최신 합의

### 모드 정의

- 제0 모드:
  - DSP 전용 또는 저부하 기본 모드
  - AI 처리가 굳이 필요하지 않거나, 과부하/오류/저전력 등으로 AI 경로 안정성이 낮을 때 사용

- 제1 모드:
  - 경량 AI 실시간 기본 모드
  - 평시 메인 모드
  - "애지간하면 실시간이 되는 가벼운 모델"이 들어가야 한다는 점을 핵심 원칙으로 유지

- 제2 모드:
  - 고성능 AI 조건부 모드
  - 포화, 고소음, 강한 왜곡 등으로 추가 복원이 실제로 필요할 때만 사용
  - 예상 처리 지연이 허용 범위 내일 때만 실시간 경로에 올리는 것이 자연스럽다

### 모드 판단 순서

- 1단계: AI가 필요한지 먼저 판단한다
  - 불필요하면 제0 모드

- 2단계: AI가 필요하면 기본값은 제1 모드로 둔다
  - 실시간 통신의 기본 책임은 경량 AI가 진다

- 3단계: 제2 모드가 정말 필요한지 추가 판단한다
  - 포화율, 소음 수준, 왜곡 심도 등 복원 필요성을 본다

- 4단계: 제2 모드가 필요해도 예상 처리 지연이 허용 범위 내일 때만 제2 모드를 선택한다
  - 만족하지 못하면 제1 모드를 유지한다
  - 필요 시 손상 구간에 한해 선택적 후속 처리 또는 보조 복원 경로를 부가 실시예로 붙일 수 있다

### 문서화 원칙

- 제1 모드는 "애매한 중간 모드"가 아니라 메인 실시간 AI 모드로 읽혀야 한다
- 제2 모드는 항상 켜지는 기본 모드가 아니라 품질 우선의 조건부 모드로 읽혀야 한다
- 후속 수정 시에는 `제0 모드 = DSP`, `제1 모드 = 경량 AI`, `제2 모드 = 고성능 AI`의 역할 구분을 흐리지 않는 것이 중요하다
- `지연 예산`이라는 용어를 고집하기보다 `예상 처리 지연`, `허용 지연 범위`, 또는 그에 준하는 평이한 표현을 우선 검토한다
- 독립항에서는 `복수의 처리 모드 중 선택 또는 전환` 정도의 뼈대를 두고, 제0/제1/제2 모드의 상세 정의와 전이 기준은 종속항 또는 상세한 설명으로 내릴 수 있다
- 제0 모드는 독립항의 필수 요소로 강조하기보다, 복수 모드 운용 구조의 한 실시예 또는 종속적 구체화로 다루는 편이 자연스럽다

### 적응형 연산 제어와 적응적 융합

- `적응형 연산 제어`는 입력 상태와 연산 여건에 따라 처리 모드를 선택 또는 전환하는 로직을 뜻한다
- `적응적 융합`은 공기전도 채널과 골전도 채널의 기여도를 조절하여 결합 또는 출력하는 로직을 뜻한다
- 후속 문서 수정 시 두 표현의 경계를 흐리지 않도록 유지한다

## 명세서/청구항 표현 가이드

### 피해야 할 표현

- 포화로 손실된 원신호를 정확히 복원한다
- 전장에서는 항상 마이크가 포화된다
- LLM이 원래 발화를 복원한다

### 권장 표현

- 비포화 채널의 정보를 우세하게 활용하여 음성 정보를 재구성한다
- 공기전도 채널이 포화되고 골전도 채널이 유효한 경우, 골전도 채널 정보를 우세하게 활용하여 음성 정보를 재구성한다
- 포화 구간의 신뢰도 저하를 반영하여 채널별 기여도를 조절한다
- 음성의 명료도 또는 통신 연속성을 확보한다
- 양 채널 동시 포화 시 복원 대신 보간, 감쇠, 자연스러운 페이드, 제한된 배경 소음 유지, 무음, 또는 경고 중 하나를 수행한다
- 양 채널 동시 포화 시 복원 대신 자연스러운 연결 또는 감쇠/무음/경고 처리를 수행한다

## 청구항 구조 권장안

- 장치 청구항:
  - 보호장구류 일체형 또는 결합형 멀티모달 음성 통신 장치
  - 골전도 센서, 공기전도 마이크 어레이, 포화 상태 정보 생성 입력부
  - 적응형 연산 제어 및 적응적 융합을 포함하는 신호처리부
  - 차음 구조체, 골전도 출력부, 통신 인터페이스
  - `대체 처리 경로`는 독립항의 병렬 핵심 요소로 고정하기보다, 모드 전환 또는 출력 제어의 하위 구체화로 둘 수 있다

- 운용 방법 청구항:
  - 포화 검출
  - 포화 상태 정보 생성
  - 모드 전환
  - BC/AC 융합
  - 대체 처리 경로 전환

- 제조/캘리브레이션 방법 청구항:
  - 센서 배치
  - 접촉압 설정
  - 전달함수 측정
  - 포화 임계값 및 회복 특성 보정

- 학습 방법 청구항:
  - 장치 물성 반영 합성 데이터 생성
  - 포화/회복시간/비선형 응답 반영
  - 포화 상태 정보를 입력 또는 제어 변수로 사용하는 모델 훈련
  - 특히 보호장구류 물성, 전달함수, 접촉압 변동, 및 포화 마스크 조건을 함께 반영하는 조건부 훈련 구성을 차별 포인트로 검토한다

- 매체/프로그램 청구항:
  - 위 제어방법을 수행하는 프로그램 또는 기록매체

## 보조 복원 경로에 대한 현재 판단

- 보조 복원 경로는 핵심 발명의 필수 구성으로 보지 않는다
- 현재 컨셉에서 핵심은 보호장구류 일체형 실시간 음성통신, 포화 대응, 멀티모달 융합, 및 대체 처리 경로다
- 현재 컨셉에서 핵심은 보호장구류 일체형 실시간 음성통신, 포화 대응, 멀티모달 융합, 및 필요 시 출력 형태를 바꾸는 대체 처리다
- 보조 복원 경로는 필요 시 손상 구간을 별도로 저장하여 후속 처리하는 선택적 부가 실시예로 두는 편이 자연스럽다
- 청구항에서는 독립항보다 종속항에서 다루는 것이 우선 권장된다

## 외부 검토 답변 활용 원칙

- 외부 검토 답변은 심사 또는 무효 리스크를 줄이는 참고자료로 활용하되, 본 발명의 정체성과 서사를 대체하지는 않는다
- 특히 `보호장구류 일체형 또는 결합형 극한소음 음성통신 장치`라는 발명의 얼굴은 약화하지 않는다
- `포화 상태 정보`를 중심 개념으로 두고, 별도의 새 중심 용어를 키우기보다 필요한 판단 보조 정보는 설명 수준에서 보강하는 현재 방향을 유지한다
- 모드 체계는 기능적으로 설명하되, 명세서에서는 `제1 모드 = 경량 AI 실시간 기본 모드`, `제2 모드 = 조건부 고성능 AI 모드`라는 운용 감각을 잃지 않는다
- 양 채널 동시 포화는 단순 무음 또는 경고만의 문제가 아니라, 거짓 음성 출력을 억제하면서 자연스럽게 망가지는 `graceful degradation` 철학까지 포함해 본다

## 후속 재검토 포인트

- 제1 모드가 실제로 메인 실시간 모드로 읽히는가
- 제2 모드 진입 조건에 복원 필요성과 지연 허용성이 모두 반영되어 있는가
- 제2 모드가 허용 지연 범위를 초과할 때 제1 모드 유지 또는 보조 복원 경로 활성화 논리가 자연스러운가
- 포화 상태 정보를 `포화 마스크` 중심으로 유지하면서 판단 보조 정보를 어디까지 설명할지
- 양 채널 동시 포화 대응을 독립항까지 넓힐지, 종속항 또는 실시예 수준으로 한정할지

## 1~7 검토 체크리스트

- 1. 양 채널 동시 포화 대응
  - 현재 기울기: 독립항의 핵심으로 올리기보다, `포화 상태 정보 기반 대체 처리 경로 전환` 수준만 상위에 두고, 구체 대응은 종속항 또는 실시예로 내리는 쪽
  - 확인할 점: `자연스러운 연결`과 `출력 전환`을 어떤 조합으로 표현할지

- 2. 제0/제1/제2 모드 정의와 전이 조건
  - 현재 기울기: 현재 합의 유지
  - 제0 모드는 DSP 또는 저부하 기본 모드, 제1 모드는 경량 AI 실시간 기본 모드, 제2 모드는 복원 필요 시에만 쓰는 조건부 고성능 AI 모드
  - 확인할 점: `지연 예산` 대신 `허용 지연 범위` 또는 그에 준하는 표현으로 충분한지, 전이 조건을 얼마나 추상적으로 쓸지

- 3. 보호장구류/헬멧 일체형 아이덴티티
  - 현재 기울기: 강하게 유지
  - `보호장구류 일체형 또는 결합형`은 상위 레벨에서 살리고, `보호 헬멧`은 대표 실시예 또는 종속항으로 받을 수 있다
  - 센서/입출력은 헬멧 중심으로 두되, 연산 모듈이 조끼, 복부 보호대, 또는 휴대형 장착체에 분산 장착되는 변형예도 허용한다
  - 확인할 점: 독립항에서 어디까지 넓히고 어디서부터 헬멧 및 분산 장착 구성을 구체화할지, 휴대형 장착체를 보호장구류 자체와 어떻게 구분할지

- 4. 포화 상태 정보와 센서 신뢰도 맵의 위계
  - 현재 기울기: 현재 합의 유지
  - `포화 상태 정보`가 중심 개념이고, 별도의 새 용어를 키우기보다 필요한 판단 보조 정보를 설명하는 쪽이 우선이다
  - 확인할 점: 포화 상태와 채널 상태 참고정보의 경계를 어떻게 정리할지

- 5. 보조 복원 경로
  - 현재 기울기: 선택적 부가 실시예 또는 종속항으로 유지
  - 핵심 발명의 필수 요소로는 두지 않는다
  - 확인할 점: 후속 처리, 비실시간 처리, 저장 구간 처리 중 어느 수준까지 남길지

- 6. 독립항과 종속항의 역할 분담
  - 현재 기울기: 독립항은 발명의 뼈대만, 종속항은 구체화
  - 독립항 뼈대 후보는 `보호장구류 적용성 + AC/BC 입력 + 포화 상태 정보 + 적응형 처리 또는 복수 모드 중 선택/전환`
  - `대체 처리 경로`는 독립항의 별도 병렬 요소로 고정하기보다, 모드 전환 또는 출력 제어의 구체 실시예로 둘 수 있다
  - 독립항에서는 `복수의 처리 모드 중 선택 또는 전환` 정도의 뼈대만 두고, 제0/제1/제2 모드의 상세 정의와 전이 기준은 종속항 또는 상세한 설명으로 내릴 수 있다
  - 확인할 점: 독립항을 몇 개 축으로 나눌지, 장치/방법/학습/제조 항목의 우선순위를 어떻게 둘지

- 7. 전체 용어 체계
  - 현재 기울기: `정확 복원` 중심 표현은 약화하고, `명료도`, `통신 연속성`, `대체 처리 경로`, `오복원 억제` 중심으로 통일
  - 다만 비포화 채널이 유효한 경우의 `재구성` 또는 제한적 `복원` 표현까지 전면 금지할 필요는 없다
  - 확인할 점: 어떤 문맥에서만 `복원`을 허용하고, 어떤 문맥에서는 반드시 피할지
